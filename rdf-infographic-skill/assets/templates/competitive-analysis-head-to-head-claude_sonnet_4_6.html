Platform Capabilities

Capabilities Breakdown

Neo4j Virtual Graph brings 4 new capabilities; Virtuoso has delivered 7 for over a decade.

◈ Neo4j Virtual Graph — 4 Capabilities (2026, private preview)
❄️
Zero-Copy Federation 2026
Data stays in Snowflake or Databricks; Cypher queries execute in place via SQL translation. No ETL, no new system of record.
⚙️
Deterministic Cypher→SQL Compilation 2026
Cypher patterns compiled to optimized SQL deterministically (not by LLM), ensuring predictable performance and consistent results.
🤖
AI-Powered Graph Schema Inference 2026
AI generates a graph schema from relational tables by inferring relationships from foreign keys and detecting implied relationships.
🔢
Graph Compute Layer 2026
Handles graph-specific operations — multi-hop traversal, path queries — that SQL cannot efficiently express.
🔮 Virtuoso Universal Server — 7 Capabilities (1998–present)
🗄️
RDF Views (Virtual RDF Graphs) ~2007
Map relational SQL tables to RDF triples without copying data. SPARQL queries execute on live relational data. Predates Neo4j VG by ~19 years.
📋
Linked Data Views (W3C R2RML) 2012
W3C-standard R2RML declarative mappings for zero-copy virtual KG construction. Formally interoperable; auditable under governance requirements.
🌐
SPARQL 1.1 Federated Queries 2013
SERVICE clause federation across any W3C SPARQL endpoint. Powers DBpedia and demonstrated with live Wikidata + DBpedia demos.
⭐
ISO GQL (ISO/IEC 39075) Support 2025
Native ISO GQL over virtualized SQL, federated Wikidata + DBpedia. Six live demos including weighted degree centrality and SPARQLWorks viz.
🔵
openCypher Support ~2022
Cypher graph pattern queries over Virtuoso's multi-model store — same syntax as Neo4j, broader data scope.
🔀
Multi-Model Simultaneous Query ~2015
SQL, SPARQL, openCypher, ISO GQL, GraphQL on the same data simultaneously. 'Loosely coupled architecture' — HTTP IRIs as universal entity identity.
🕷️
Virtuoso Sponger Middleware ~2007
Real-time RDF extraction from web content via extraction cartridges. Virtual KG over any structured/semi-structured/HTML content without ETL.
Open Standards Landscape

Standards Support

Six key open standards — and which platform supports each today vs. on roadmap.

📡
W3C SPARQL 1.1
✓ Virtuoso — full ◯ Neo4j — not targeted
W3C graph query + federation standard. SERVICE clause enables cross-endpoint federated queries.
📐
W3C RDF 1.1
✓ Virtuoso — native store ◯ Neo4j — not supported
Foundation data model using subject-predicate-object triples with HTTP IRIs for entity identity.
🗺️
W3C R2RML
✓ Virtuoso — Linked Data Views ◯ Neo4j — AI inference instead
Declarative RDB→RDF mapping. Enables zero-copy virtual KG construction that is formally interoperable.
⭐
ISO/IEC 39075 GQL
✓ Virtuoso — implemented ⏱ Neo4j — roadmap
International standard for graph query. Both platforms converging — Virtuoso already delivered; Neo4j planning.
🔵
openCypher
✓ Virtuoso — supported ✓ Neo4j — primary language
Open spec for Cypher graph query language. The common ground — supported by both platforms.
🔗
W3C Linked Data
✓ Virtuoso — native HTTP IRIs ◯ Neo4j — internal node IDs
HTTP IRIs as global entity identifiers with content negotiation. Enables cross-system semantic interoperability.
